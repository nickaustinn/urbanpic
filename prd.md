# UrbanPic — Product Requirements Document

**Version:** 1.0
**Date:** March 2026
**Team:** nickaustinn, mollindabo
**Hackathon:** 2-day sprint

---

## 1. Problem Statement

City residents regularly encounter civic issues — potholes, graffiti, broken streetlights, illegal dumping — but reporting them through official 311 systems is tedious and inconsistent. Forms are long, department routing is unclear, and most residents simply don't bother. As a result, issues go unreported, neighborhoods deteriorate, and city departments operate without an accurate picture of where problems are concentrated.

## 2. Goals

| # | Goal | Success Metric |
|---|------|----------------|
| 1 | Reduce friction in civic reporting | User can submit a report in < 30 seconds |
| 2 | Improve report quality via AI | 90%+ of AI-generated descriptions are accepted without editing |
| 3 | Route to correct department automatically | 85%+ routing accuracy on test set of 20 images |
| 4 | Surface geographic patterns | Heat map renders all reports with < 1s load time |
| 5 | Demo-worthy in 2 days | End-to-end flow works live at demo |

## 4. User Flow

```
1. Landing page
   └─ "Report an Issue" CTA

2. Photo capture step
   ├─ Upload photo from device
   └─ Take photo with camera (mobile)

3. AI analysis (loading state)
   └─ Claude Vision classifies issue type + generates description

4. Review & confirm step
   ├─ Show AI classification (editable)
   ├─ Show generated 311 description (editable)
   ├─ Show routed department
   └─ Pin location on mini map

5. Submit
   └─ POST /api/reports → save to DB

6. Success screen
   ├─ Confirmation + report ID
   └─ Link to community heat map

7. Heat map page
   └─ All reports as clustered pins on Mapbox map
```

## 5. Out of Scope (v1)

- Native mobile app
- Real city 311 API integration
- SMS/email notifications
- Admin dashboard for city employees
- Duplicate detection across reports

## 6. Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|
| Claude misclassifies edge-case images | Medium | Medium | Allow user to override classification |
| Mapbox quota exceeded during demo | Low | High | Cache map tiles, use free tier conservatively |
| Database not set up in time | Low | High | Fallback to in-memory store for demo |
| Mobile camera API inconsistencies | Medium | Low | Prioritize file upload, camera as enhancement |
| Running out of Anthropic API credits | Low | High | Add rate limiting, use haiku model as fallback |
| Scope creep during sprint | High | Medium | Strictly enforce P0-only until Saturday afternoon |

## 7. Non-Functional Requirements

- Page load time < 2s on 4G mobile
- AI classification response < 5s
- Works on Chrome, Safari, Firefox (latest)
- Mobile-first responsive design
- No PII stored beyond what's needed for the report
